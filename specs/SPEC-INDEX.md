## Clause ID Grammar

```
clause_id := "PNP-" spec_num "-" level "-" serial
spec_num  := 3 ASCII digits
level     := "MUST" | "SHOULD" | "MAY"
serial    := 3 ASCII digits, unique within (spec_num, level)
```

Clause IDs are stable. Once a clause ID is published in a CANDIDATE or ADOPTED spec it MUST NOT be recycled, reassigned, or silently removed. If a clause is superseded it is marked `[DEPRECATED]` in-place and the replacement receives a new serial.

## Status Lifecycle

| Status | Meaning |
|--------|---------|
| DRAFT | Incomplete or actively restructuring; not suitable for conformance testing. |
| CANDIDATE | Clauses stable enough to be pinned by conformance tests. Changes require version bump + changelog. |
| ADOPTED | Ratified with ≥ 100% MUST coverage and ≥ 80% SHOULD coverage; wire-incompatible changes forbidden. |
| SUPERSEDED | Replaced by a later PNP; retained for historical reference. |
